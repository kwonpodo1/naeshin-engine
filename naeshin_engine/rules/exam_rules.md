## 0. 출제 대원칙 (KICE·교육청 공식 기준)

모든 변형문제는 아래 3대 원칙을 **최우선**으로 지킨다. 위반 시 복수정답·정답없음 시비가 발생하며, 이는 시·도교육청 학업성적관리 지침상 **징계 대상**이다.

1. **오답 제1원칙** — 오답(distractor)은 본문 주제·내용과 *관련되되* **"또 다른 정답"이 되어선 안 된다.** 정답의 동의어/유의어/반의어를 오답으로 쓰면 복수정답이 된다. (어휘 오답: 목표어와는 가깝게, 정답과는 멀게)
2. **정답 유일성** — 순서·삽입·도표 유형은 정답이 **하나만** 나오도록 설계한다. 가급적 지시어·연결어보다 **내용상 의미관계**로 풀리게 한다.
3. **출제 금지** — ①참고서 문제 전재/일부 변경 ②기출 그대로 재출제 ③복수정답·정답없음 문항. (`run_quality_check.py`가 **선지 중복(복수정답 위험)**을 자동 점검하고, 의미적 복수정답은 **09 SEMANTIC-VERIFIER**가, ①②는 출제자가 확인)

## 2. 섹션별 발췌 방식 (핵심 규칙)

### 발췌 원칙
- 지문 전체를 한 번에 쓰지 않는다
- 지문을 **의미 단위**로 나눠 각 섹션별로 발췌 지문 + **1~2문제** 세트 구성
- **중복 방지 원칙**: 각 문제에 사용되는 발췌 지문 덩어리들이 서로 내용상 최대한 겹치지 않도록 완전히 독립된 구역으로 분할할 것. 애매하게 앞뒤 문장이 겹치지 않도록 주의한다.
- 발췌 기준: 하나의 주제/섹션이 시작~마무리되는 자연스러운 단위
- 발췌 지문 분량: **약 400\~600자** (너무 짧으면 문맥 파악 불가, 너무 길면 컬럼 초과)
- 총 분량: 사용처 정책에 따름 (auto 회당 30문항 / studio 가변 — 분량은 각 제품이 지정)

### 지문 가공 규칙 (문제 유형별)

| 문제 유형 | 지문 가공 방법 |
|---------|-------------|
| 빈칸_연결어 / 빈칸_어휘 / 빈칸_문장 | 해당 어구를 삭제하고 `(A)_____`, `(B)_____` 또는 `_______` 로 대체 |
| 어법_밑줄 | 밑줄 포인트 5곳에 `①단어` `②단어` 형식으로 번호 직접 삽입 |
| 어법_ABC | 해당 어구를 `(A)[선택1 / 선택2]` 형식으로 대체 |
| 순서_배열 | 지문을 `주어진 문장 + (A) ... (B) ... (C) ...` 블록으로 분리 표시 |
| 문장_삽입 | 지문 내 삽입 위치에 `① ② ③ ④ ⑤` 번호 직접 삽입 |
| 주제/요지/제목/내용일치/불일치/지칭추론/답할수없는것 | 지문 가공 없음 (원문 그대로) |
| 조건_영작 | 지문 가공 없음, 문제 아래 조건 박스 + 답란(밑줄 여러 줄·넉넉한 여백) |

### 출제 검수 — 흔한 결함 방지 (필수)

실제 검수에서 반복 발견된 **문항 불성립·본문 훼손** 결함과 방지 규칙:

1. **문장삽입 — 삽입 대상 문장은 본문에서 반드시 제거한다.** 주어진(삽입할) 문장이 본문 정답 위치에 그대로 남아 있으면 정답이 노출되고 문항이 성립하지 않는다. 본문에서 그 문장을 빼고 자리마다 `①~⑤` 위치 표시만 남긴다. (같은 지문의 다른 삽입 문항과 대조해 일관성 확인)
2. **어법·어휘 밑줄 — 의도한 오답(틀린 단어)을 실제로 지문에 심는다.** 밑줄 `①~⑤`가 전부 원문 그대로의 올바른 단어이면 "틀린 것이 없어" 정답이 성립하지 않는다. 정답 위치 1곳에 반드시 비문법/문맥 부적절 단어를 넣는다. **다른 문항과 지문을 공유**하는 경우, 그 문항 전용 가공 지문(`passage_underlined`)에 밑줄+오답을 담아 공유 지문(정답형)과 충돌하지 않게 한다. 어법 A/B/C는 `passage_abc`에 선택지를 담는다.
3. **지칭추론 — 밑줄 대상을 지문에 표시한다.** 대상이 **어구**이면 발문에 따옴표로 인용하고(예: `밑줄 친 "reset the clock"`) 생성기가 지문에서 자동 밑줄 처리한다. 대상이 **한 단어**(`which` 등)이면 지문에서 그 단어를 `<u>which</u>` 로 직접 밑줄 친다. 발문에 `(밑줄: …)` 같은 메모를 넣지 않는다.
4. **발문 군더더기 금지.** 표준 발문에 불필요한 보조 설명(예: `(단, 주어진 글은 결론에 해당한다.)`)을 붙이지 않는다. 표준 문구만 사용한다.
5. **본문 충실 — 발췌 지문의 원문 단어를 임의로 바꾸지 않는다.** 의미가 정반대가 되거나(예: `ceasing`→`causing`) 비문이 되어 학생이 읽는 본문이 훼손된다. 단복수·관사·연결어도 원문대로 옮기고, 의도적 가공(빈칸/밑줄/선택지)만 표시한다.

> 정답표는 서술형을 포함한 **모든 문항 번호**를 표시한다(서술형은 답란에 `서술형`). 영작 문항은 조건 박스 아래 **답란(밑줄 여러 줄)**을 둔다. — 생성기(`make_exam_pdf.py`)가 자동 처리.

## 4. 어법 문제 출제 포인트 (고등학교 내신 기준)

### 어법 문항 설계 원칙 (KICE 매뉴얼, `KB-02` A3)

- **macro > micro**: 단순 암기(micro)보다 담화·문맥에 의존하는(macro) 항목을 묻는다.
- **중복 측정 금지**: 한 문항에서 같은 문법 항목을 두 번 묻지 않는다.
- **문맥 풀이 가능**: 문맥 정보로 판단할 수 있는 항목을 비교적 짧은 지문으로 출제한다.
- **빈출 우선순위**: 어법의 약 50%가 **관계사 + 동사 변형**(수일치·시제·태·준동사 형태)이다 → 우선 검토. (단 가정법·도치·가주어진주어는 내신 빈출이나 수능엔 드묾)

### (A)(B)(C) 고르기 유형에 자주 나오는 포인트

| 포인트 | 예시 쌍 |
|--------|---------|
| 관계사 선택 | who / which / that / what |
| 분사 선택 | -ing(능동) / p.p.(수동) |
| to부정사 vs 동명사 | to-V / -ing |
| 형용사 vs 부사 | careful / carefully |
| 수 일치 | is / are |
| 능동 vs 수동 | V / be p.p. |
| be + 형용사 vs 동사원형 | interesting / interest |
| allow/help/make + O + V | to-V / V원형 구분 |
| 동격 that vs which | fact **that** / **which** (앞 절 부연) |
| in that vs 관계대명사 that | in that (~라는 점에서) / 관계대명사 that |
| 접속사 vs 전치사 | although 주어+동사 / despite + 명사(구) |
| given vs considering | given + 명사 / considering + 명사 |

### 밑줄 친 ①~⑤ 중 틀린 것 유형에 자주 나오는 포인트

| 포인트 | 오류 예시 |
|--------|---------|
| 전치사 + 동명사 | invest time into **learn** → **learning** |
| to부정사 vs 동명사 | dedicated to **work** → **working** |
| 관계사 격 오류 | the man **whom** spoke → **who** spoke |
| 분사 방향 오류 (능동/수동) | the **boring** student → **bored**; shared by the police **sharing** → **shared** |
| 수동태 오류 | was **risen** → was **raised**; hang 자동사 / hung 타동사 |
| 동명사 주어 + 본동사 수 일치 | Having a lazy brain **to allow** → **allows** |
| 형용사 vs 부사 | **careful** think → **carefully** think |
| try + to-V vs try + -ing | 의미 차이 (~을 시도 / 시험 삼아 해보다) |
| 명사절 접속사 that vs what | that + 완전한 문장 / what + 불완전한 문장 |
| 명사절 접속사 if vs that | if (~인지 아닌지) / that (~라는 것) |
| 간접의문문 어순 | where **was Compean** → where **Compean was** |
| 지각/사역동사 목적보어 | see + O + V원형/-ing / have + O + p.p. |
| 병렬구조 어긋남 | examining ... and **inform** → **informing** |
| 과거완료 시제 | before 절에서 ago 금지, had p.p. 필요 |
| 도치 후 주어-동사 수 일치 | Not only **is** he (도치) |
| for + 의미상주어 + to-V vs of + 의미상주어 | character 기반 (kind of him to-V) / 평가 (important for him to-V) |
| 가정법 시제 짝 | if절·주절 시제 불일치 (가정법 과거인데 주절 will → would) |
| 도치 후 수일치 | 부정어 문두 도치 시 `조동사+주어` 어순·수일치 (Not only **is** he ~) |
| 비교 수식어 | `very + 비교급` → much/far/even/still (very는 원급만 수식) |

> 고급 구문(가정법·도치·강조·비교)의 어법 공식·전환 1차 출처: `docs/research/KB-04`.

### 동사 핵심 어법 표준 카탈로그 (KB-05 — 내신+수능 공통 최빈출)

> 어법의 약 50%인 **동사 변형**(시제·태·수일치·준동사 형태)의 ★최빈출 함정. 공식·근거: `docs/research/KB-05`. (KB-04=고급구문 심화·상위권 변별 / **KB-05=핵심·공통 최빈출**)

- **시제**: ① `since`/`for`→현재완료 vs `ago`/특정 과거시점→단순과거(`*have p.p. ~ ago` 불가) ② 시간·조건 **부사절**은 현재시제로 미래 대용(will ✗) — 단 **명사절·형용사절이면 will ○**(절 기능으로 가름) ③ 역사적 사실=항상 단순과거 / 불변진리=시제일치 예외(현재) ④ 상태동사(know/believe/belong) 진행형 불가(단 have=먹다·think=숙고 등 의미 바뀌면 가능).
- **태**: ① 자동사(occur/happen/appear/consist of/exist) 수동 불가(`was occurred` ✗) ② 5형식 수동 시 사역·지각 OC 동사원형→**to부정사 부활**(`was made to apologize`)·사역 have/let 수동 불가 ③ that절 수동 시제단차→`S is said to have p.p.` ④ be known **to**(~에게)/**for**(유명)/**as**(자격) ⑤ 구동사 수동 전치사 유지(`be taken care of`).
- **수일치**: ① 핵 주어(동사 직전 명사 착각 금지)·구/절(동명사·명사절) 주어=단수 ② **the number of(단수) ↔ a number of(복수)** / 부분사 of+N은 N에 일치 / one of the 복수N=단수 ③ 상관접속사는 **후항 B에 일치**(`B as well as A`도 B(앞항)에) / both A and B=복수 ④ every·each·측정단위(`Ten years is`)=단수 / 도치는 뒤 진주어에 / 관계사절은 선행사에.
- **준동사 형태**(어법 최빈출): ① **본동사 vs 준동사 자리** — 한 절 정동사 하나, 목적어 有無로 능/수동·본동사 有無로 준동사 판별(해석 말고 **구조분석**) ② to만(want/decide/refuse)·-ing만(avoid/finish/suggest/consider) / remember·stop·try+to-V vs -ing 의미차 ③ 분사 뒤 목적어 有→-ing/無→p.p. / 감정분사 boring(유발) vs bored(느낌) ④ 사역·지각 OC=동사원형(`made him do`; to부정사 ✗) / `get+O+to-V` / 수동관계는 p.p.(`had my hair cut`) ⑤ 의미상주어 `for` / 인성형용사 `of` ⑥ **be used to -ing(익숙) ↔ used to V(과거습관)** / look forward to·object to + -ing(to=전치사).

> ⚠️ **이견 주의**(KB-05 B절): "send 두 수동 가능"은 출처 이견 → 단정 출제 금지 / 미래표현 will·be going to는 어휘·뉘앙스 위주(순수 어법 빈도 낮음).

### 어법 해설 작성 원칙 (참고자료 학습 반영)

어법 문제 해설은 단지 자료의 Part 4 스타일처럼 **체계적이고 명확하게** 작성:

1. **해당 어법이 무엇인지** 한 줄로 명시 (예: "과거분사: 수동과 완료 의미")
2. **왜 맞는지** 구조적 근거 제시 (예: "=which was named (관계대명사+be 생략)")
3. **오답 형태를 `[~X]` 로 표시** (예: `[naming X]`, `[to shout X]`)
4. 혼동 포인트는 **★ 주의**로 강조 (예: "★ 관계대명사 that은 계속적 용법에 올 수 없다")

## 9. 글 구조 분석 원칙 (주제/요지/제목 문제 설계용)

참고자료 학습을 통해 다음 패턴으로 지문을 분석한 후 주제·요지·제목 문제 설계:

### 글 전개 구조 파악

모든 지문은 보통 다음 4가지 흐름 중 하나:

| 구조 | 전형적 단락 순서 |
|------|--------------|
| **도입-본론-결론** | 도입부 → 본론(1) → 본론(2) → 결론 (=요지) |
| **대조형** | 과거/기존 → 대조 접속사(But/However) → 현재/새 관점 (=요지) |
| **원인-결과형** | 원인 제시 → 결과 → 결론 (= 요지) |
| **질문-답변형** | 문제 제기 → 설명 → 해답 제시 (=요지) |

### 주제/요지 판단 기준

1. **도입부 = 주제** 인 경우가 가장 많음 (전체 글의 방향 제시)
2. **결론 = 요지** 인 경우가 많음 (특히 Therefore/Thus/In short 등 뒤)
3. **대조 접속사 뒤 내용이 요지**: But/However/Yet 뒤에 글쓴이 주장
4. **질문문 바로 뒤 답변 = 주제**: "Why ~?"/"How ~?" 뒤의 답이 핵심

### 글 구조 활용한 문제 설계

- **주제_파악 / 제목_파악**: 각 단락 소제목(짧은 명사구) 뽑은 뒤 이를 통합하는 상위 개념 추출
- **무관한 문장**: 글의 흐름(도입→본론→결론)에서 벗어난 문장 = 주제와 직접 관련 없는 디테일
- **문장_삽입**: 지시어(This/These/Such)나 연결어(However/Also)로 앞뒤 관계 판단
- **순서_배열**: 대명사 지칭, 연결어, 시간/논리 순서로 판단

### 담화 응집성·논리 단서 (KB-06 — 빈칸/순서/삽입 3점 핵심)

> 빈칸·순서·삽입의 풀이·출제 단서. 공식·근거: `docs/research/KB-06`(Halliday & Hasan 응집성 + 한국 현장 합의). KB-04/05=문장 내부 구문 / **KB-06=문장 간 담화·논리**.

- **연결사 의미·논리방향**: 첨가/비교/예시/인과/요약(+, 같은 방향) · 대조역접/양보(−, 전환) · 환언(=, 등가). `however` 뒤=앞의 반대 / `for example` 앞=일반진술 / `therefore` 앞=원인 / `in other words` 뒤=앞의 paraphrase.
- **빈칸 논리 단서**(정답=본문 단서로 환원): ① 재진술(`in other words` 뒤=정답 동의) ② **대조(`however` 뒤 빈칸=앞의 반대, 최빈출)** ③ 예시 일반화(빈칸=뒤 예시들의 상위개념) ④ 인과(`therefore` 뒤=앞의 결과) ⑤ **통념-반박(not A but B: 통념 뒤 `but`=필자 주장)** ⑥ 위치별(첫=주제 / 중간=다음 문장에 무게(단 예시면 계속) / 마지막=앞 문장). → 정답은 반드시 본문 단서로 환원, 오답은 (a)본문 미언급 (b)방향 반대 (c)지엽 사실(정답 유일성, 0절·5절 연동).
- **순서·삽입 응집 단서 5종**: ① 지시어(this/these/such+명사 — 선행 대상 앞에 필수) ② 관사(신정보 a→구정보 the; ⚠️단독 단서 금지) ③ 연결사(논리 강제) ④ 대명사 지칭(선행사 앞 필수) ⑤ 어휘 응집(반복/유의어/상위어 사슬). **순서**: 첫 단락=지시어·대명사·역접 없는 단락 / 소거법("붙일 수 없는 것" 기준). **삽입**: 박스 문장부터 읽어 연결사·지시어로 후보 좁힘 → 논리적 단절(gap)·선행사 없는 대명사/정관사 갑툭 지점이 정답.

> ⚠️ **고난도화 주의**(KB-06 D): 평가원은 접속사·대명사를 의도적으로 빼서 표면 단서만으론 안 풀리게 한다 → 변형문제도 **어휘 응집+내용 논리의 유일성**까지 확보해야 실제 수능 난도(표면 단서만 의존 시 너무 쉬워짐). "however 80% 추론" 등 경험 수치 인용 금지.

## 10. 어휘 문제 오답 설계 원칙

참고자료(단지 5파트, EBS 올림포스)에서 학습한 어휘 문제 제작 원칙:

### 어휘_밑줄 (적절하지 않은 단어) 오답 설계

5개 선택지 중 1개만 문맥에 어긋나게:

| 패턴 | 설명 |
|------|------|
| **반의어로 교체** | `increases` → `decreases` (가장 빈출 유형) |
| **유사어이지만 뉘앙스 차이** | `minimizes` vs `highlights`, `ignore` vs `expose` |
| **연어(collocation) 파괴** | `raise a question` → `rise a question` 같은 어색함 |
| **긍정/부정 극성 반전** | `beneficial` → `harmful`; `success` → `failure` |

### 빈칸_어휘 오답 설계

5개 중 1개만 정답, 나머지 4개는:

1. **반의어**: 정답의 정반대 의미
2. **부분 일치**: 문맥 일부에만 맞음 (전체 지문 흐름과는 어긋남)
3. **지문 내 등장 단어**: 본문 다른 부분에 나왔지만 빈칸에는 부적절
4. **모호한 상위어**: 너무 일반적이어서 구체성 부족
5. **paraphrasing 함정**: 같은 의미 같아 보이지만 글의 핵심어가 아님

### 변형 어휘 (동의어/반의어) 활용

단지 자료 Part 3처럼 **정답 단어의 동의어·반의어를 3~5개 이상 알고 있어야** 좋은 오답 설계 가능:

- 예시 어휘 풀:
  - `strategic` = tactical / systematic / calculated / deliberate
  - `continuity` = survival / persistence / permanence
  - `adaptive` = flexible / responsive / adjustable
  - `die out` = disappear / vanish / become extinct
  - `available` = accessible / obtainable / attainable

### 다의어 주의

다의어는 **지문 문맥의 의미가 무엇인지** 명확히 파악한 후 오답 설계:

- 예: `condition` → 환경(context) / 건강상태(health) / 조건(term) / 상태(state)
- 오답은 같은 다의어의 **다른 의미**로 유도

### 어휘 distractor 설계 종합 (KB-07 — 어원·혼동어·다의어 재료)

> 어휘 오답 풀의 체계. 공식·근거: `docs/research/KB-07`. **상위 원칙은 0절·5절 오답 제1원칙(또 다른 정답 금지)** — KB-07은 그 아래 "어떤 단어를 오답으로 쓸지"의 재료.

| distractor 유형 | 설계법 |
|---|---|
| **반의어 치환**(★1순위) | 정답의 명확한 반의어(faster→slower·increase→decrease). 수능 어휘(30번) 정답 단어의 표준 함정. |
| **혼동어 치환** | 형태유사 다른 단어(effect↔affect·raise↔rise·adapt↔adopt·principal↔principle) |
| **같은 어근 접미사 변형** | imaginary/imaginative/imaginable·considerate/considerable·economic/economical (어휘 변형 빈칸의 "또 다른 정답 없는" 오답 세트) |
| **다의어 1차 뜻 함정** | 파생 의미가 정답일 때 표면 뜻을 오답(address=주소를 오답, "다루다"를 정답) |
| **⚠️ 억지 반의어 금지** | 부자연스러운 부정·반의(caused→"야기 안 됨")는 복수정답 시비 → 사용 금지 |

- **어휘 출제 방식별**: 밑줄 적절성(반의어 치환) / 문맥 함축 추론(오답=표면적 1차 해석) / 어휘 변형(어근동일 접미사군) / 영영풀이 매칭(오답=혼동어 정의 섞기).
- ⚠️ "한국어 반의어 떠오르면 정답"은 수험 휴리스틱(평가원 공식 아님) — **정답 유일성을 상위 원칙으로** 유지.

## 11. 서술형/논술형 문제 설계 (한 문장 요약 / 조건 영작)

### 한 문장 요약 (논술형 예상)

EBS 올림포스 아잉카 스타일처럼 지문 핵심을 **영어 한 문장 + 한국어 한 문장**으로 작성:

```
* 한 문장 요약 출제 예상

The incorporation of scientific methodology into medicine reinforced
its professional authority and elevated its credibility, ultimately
altering the fundamental structure of doctor-patient relations.

과학적 방법론의 의학 도입은 의학의 전문적 권위를 강화하고 신뢰성을 높였으며,
궁극적으로 의사-환자 관계의 근본적인 구조를 변화시켰다.
```

**작성 원칙**:
- 지문 전체 주제 + 핵심 논지 포함
- 원문의 **핵심 단어**를 유지하되 문장 구조는 재구성 (paraphrasing)
- 25~40 단어 수준 (너무 짧으면 내용 누락, 너무 길면 요약이 아님)

### 조건 영작 설계

조건은 **서술형 예상 문장(star=True)에서 선발**한 구문을 활용:

- 서술형 예상 기준(RULES.md 6번)에서 뽑힌 문장을 우선 조건 영작의 원문으로 사용
- 조건에는 **해당 문법 포인트를 강제**: "동명사 주어로 시작할 것", "It ~ that 강조구문을 사용할 것"
- 주어진 단어에는 **정답을 맞추려면 반드시 변형해야 하는 단어** 포함 (예: "exercise" 줄 때 정답에서 "exercising"이 되도록)

### 서술형 채점 루브릭 9요소 (KERIS 공식, `KB-02` A5)

서술형/조건영작 문항의 채점 기준은 아래 요소로 구성한다(데이터에 필드로 담아 자동 생성 가능):

| 요소 | 설명 |
|------|------|
| 예시답안 | 기본/모범 답안 |
| 인정답안 | 유사·동등 표현 허용 범위 |
| 배점 | 문항 총점 |
| 채점영역 | 채점 요소(내용/구조/어법 등) |
| 채점기준 | 영역별 정오 판정 기준 |
| **부분점수** | 영역별 중요도 차등·단계 배점 (예: 근거 3가지↑=3 / 2가지=2 / 1가지=1) |
| 유의사항 | 채점자 간 일관성 확보 메모 |

- **감점 규칙**: 철자·띄어쓰기 같은 기계적 오류는 비중을 축소하고, **맞춤법 감점은 횟수와 무관하게 1회만**(감점이 총점을 잠식하지 않게).
- **채점 타당성**: 포괄성(모든 답안에 대응 기준 존재) + 배타성(점수 구간이 겹치지 않음).
- **발문 구체화**: 모호한 광범위 지시(✗ "지구온난화에 대해 설명")가 아니라 답할 방향을 구체화한다(○ "원인을 설명하고, 문제점과 해결방안을 설명"). (`KB-02` A6)
